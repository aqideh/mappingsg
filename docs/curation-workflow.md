# Curation workflow

## Submission intake

Every community memory enters the archive as a pending record. Public publication is not automatic.

Required checks:

1. The contributor has agreed that the memory can be published.
2. The contributor has confirmed that uploaded media can be used.
3. Curators have checked whether the memory identifies living people.
4. Location has been verified or marked approximate.
5. Tags and themes use the approved vocabulary.
6. Audio has a transcript or summary for accessibility.
7. Images have captions and alt text.

## Review decisions

Curators can choose one of these outcomes:

- Publish: record is visible on the public map.
- Needs revision: contributor or curator must update the record.
- Archive: retain internally but hide from public map.
- Reject: do not retain as an archival record.

## Public visibility rule

A memory is public only when all of the following are true:

- status is published
- can_publish is true
- rights_confirmed is true

## Sensitive records

Records involving minors, family disputes, medical details, financial hardship, or identifiable third parties should receive additional review before publication.

## Recommended admin roles

- Contributor: can submit memories.
- Curator: can review and edit memory metadata.
- Admin: can manage tags, overlays, and publication settings.
